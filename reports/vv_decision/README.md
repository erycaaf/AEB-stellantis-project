# Decision (TTC + FSM) V&V — Evidence artefacts

Reproducible evidence for the independent cross-validation of
[src/decision/aeb_ttc.c](../../src/decision/aeb_ttc.c) and
[src/decision/aeb_fsm.c](../../src/decision/aeb_fsm.c).
**Validator (cross):** Eryca. **Author:** Lourenço Jamba Mphili.
Aligned to the ISO 26262-6:2018 ASIL-D technique set.

The consolidated V&V report (`Consolidated_Report_VV_Decision.pdf` and its
`.tex` source) lives in the team documentation area, **outside this repo**.
Only the machine-generated evidence and the reproducibility build targets
live here.

## Reproduce everything

From the repo root, in an environment with GCC 14, gcov-14, gcovr 8.6,
cppcheck 2.13, Valgrind 3.22 and the ASan/UBSan runtimes (Ubuntu 24.04
provides all of these):

```bash
make vv-decision
```

This runs, in order:

1. `make mcdc-decision` — `-fcondition-coverage` build + gcov condition report
2. `make fault-decision` — systematic fault-injection suite (non-fatal harness)
3. `make memory-decision` — Valgrind + ASan + UBSan on all three test binaries
4. `make misra-decision` — cppcheck MISRA addon scoped to decision module

## Robustness vulnerabilities (from fault injection)

Four defects identified by the fault-injection activity; root-cause
analysis and proposed patches are in the consolidated V&V report. Per
ISO 26262-6 §5.4.8 (independence), the patches are to be applied by the
original author on a separate branch and re-validated by re-running
`make vv-decision`.

| # | Severity | Location | Evidence (this repo) |
|---|---|---|---|
| 1 | HIGH (robustness) | `src/decision/aeb_ttc.c:40-54` | `fault_injection/run.log` (A4, A5, B2) |
| 2 | CRITICAL (SEU) | `src/decision/aeb_fsm.c:175` | `fault_injection/run.log` (C3) |
| 3 | HIGH (robustness) | `src/decision/aeb_fsm.c:165` | `fault_injection/run.log` (A6) |
| 4 | CRITICAL (SEU) | `src/decision/aeb_fsm.c:230, :328` | `fault_injection/run.log` (C1, C2) |

The MC/DC coverage activity additionally flagged 3 defensively unreachable
lines (dead code per MISRA C:2012 Rule 2.1) — removal proposal in the
consolidated V&V report.
